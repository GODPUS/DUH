Breakpoints

in _variables.scss:

key is the name, first value in list is the min-width of the screen to trigger the breakpoint. second value in the list is the width of the .container class at that screen size. the breakpoint names will be used as the suffix on responsive classes.
$breakpoints: ('sm': (0, 100%), 'md': (768, 100%), 'lg': (960, 960px));

Respond to breakpoints with sass using either:

in this example the and-up mixin will apply rules to size md and up (applying only to md and lg. ignorning sm)
@include and-up(md){ border: none; }
or
in this example the only mixin will only apply rules to the specified size (applying only to md. ignorning sm and lg.)
@include only(md){ border: none; }

Respond to breakpoints with javascript using:

$(strapOn).on('breakpoint', function(){ console.log(strapOn.breakpoint); });

Columns

column numbers = 100%/column number.
col-1-sm = 100%;
col-2-sm = 50%;
col-3-sm = 33.3%;
col-4-sm = 25%;
etc...
100% = .col-1-sm 50% = .col-2-md 33.3% = .col-3-lg
50% = .col-2-sm 33.3% = .col-3-md 100% = .col-1-lg
33.3% = .col-3-sm 100% = .col-1-md 50% = .col-2-lg

Gutters

use gutter classes in conjunction with row classes to specify the gutter space for direct column children
$gutterSizes: (0, 5, 10, 15, 20, 25, 30);
the default gutter size of col- classes
$defaultGutterSize: 10;

Javascript via data attributes

a combination of data-show-href, data-hide-href, data-toggle-href, data-scroll-href, data-tab-group are used to make dropdowns, accordions, tabs, modals, and scroll-to's

Dropdowns

  • if two elements have the same href they will update the other when one changes like with this select group and button group
  • a 'show' and a 'hide' event are triggered for both the button and the href, which you can listen and respond to via js like in demo.js
  • an .active class is also added to both the button and the href which you can use via css for styling and cssanimations
  • add a data-tab-group to all grouped buttons so that they are unselected when another is selected
